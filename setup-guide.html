Xemelgo Inventory API

Power BI Setup Guide

Run the Xemelgo inventory Python script directly inside Power BI — no CSV, no external files.

1
Download Python

Click the link below to download the Python installer directly. It will download a file called python-3.14.5-amd64.exe to your Downloads folder.

Download Python 3.14.5 for Windows (64-bit)

ℹ️
Not sure if you're on 64-bit Windows? You almost certainly are — any computer made after 2012 is 64-bit. If you're unsure, this version is safe to use.
2
Run the installer — IMPORTANT: check the box at the bottom

1. Open your Downloads folder and double-click python-3.14.5-amd64.exe

2. A window will appear. Before you click anything else, look at the very bottom of the window — you will see a checkbox that says "Add python.exe to PATH"

⚠️
Check that box. This is the most important step. If you skip it, nothing will work and you'll need to reinstall. The box is NOT checked by default.

3. After checking the box, click "Install Now" (the top option)

4. If Windows asks "Do you want to allow this app to make changes to your device?" — click Yes

5. Wait for the progress bar to finish (about 1–2 minutes)

6. When you see "Setup was successful", click Close

3
Verify Python installed correctly

1. Press the Windows key on your keyboard (the key with the Windows logo, bottom-left of your keyboard)

2. Type cmd and press Enter. A black window called "Command Prompt" will open.

3. Type the following exactly and press Enter:

shell 
python --version

You should see: Python 3.14.5

⚠️
If you see an error like "'python' is not recognized" — it means you forgot to check the PATH box in Step 2. Uninstall Python via Windows Settings → Apps, then redo Step 2 and make sure to check the box.
4
Install the required packages

Still in the black Command Prompt window from the previous step (or open a new one — Windows key → type cmd → Enter), type the following exactly and press Enter:

shell 
pip install requests pandas matplotlib

You will see a lot of text scroll by — that is normal. Wait until it stops and you see a new line with a blinking cursor.

Then run this to confirm everything installed:

shell 
python -c "import requests, pandas, matplotlib; print('All good!')"

You should see: All good!

💡
You can close the Command Prompt window now. You won't need it again.
5
Tell Power BI where Python is installed

Open Power BI Desktop. If you don't have it, download it free from microsoft.com/en-us/download/details.aspx?id=58494

Then:

1. Click File in the top-left corner

2. Click Options and settings

3. Click Options

4. On the left side panel, under the Global section, click Python scripting

5. Under Detected Python home directories, open the dropdown — if you see a Python path listed, select it. Most of the time Power BI finds it automatically after installation.

6. If it shows "Other" or nothing useful, open a new Command Prompt (Windows key → type cmd → Enter) and run:

shell 
python -c "import sys; print(sys.executable)"

You'll see something like C:\Users\YourName\AppData\Local\Programs\Python\Python314\python.exe. Copy everything except the \python.exe at the end, and paste that into the Power BI field.

7. Click OK

8. Close and reopen Power BI Desktop so it picks up the change.

6
Load your inventory data into Power BI

In Power BI Desktop:

1. Click Home in the top ribbon

2. Click Get Data

3. In the search box that appears, type Python

4. Click Python script in the results, then click Connect

5. A text box appears. Click inside it, select all with Ctrl+A, delete it, then paste the script below

6. Before clicking OK, find the two lines near the top that say EMAIL and PASSWORD and replace the placeholder text with your actual Xemelgo login credentials

7. Click OK — Power BI will run the script (this may take 30–60 seconds depending on how many items you have)

python 
import base64, json, requests, pandas as pd

EMAIL    = "your@email.com"
PASSWORD = "yourPassword"
FILTER   = None   # e.g. 'not isConsumed', or None for all items

LOGIN_URL   = "https://rest.api.xemelgo.com/login"
GRAPHQL_URL = "https://api.xemelgo.com/graphql"

QUERY_TEMPLATE = """query {
  inventory(input: { filter: %s, nextToken: %s }) {
    inventory {
      id name description trackerSerial state isConsumed
      lotNumber customerPartNumber creationDate lastUpdatedDate
      expirationDate consumedDate customProperties
      part { id number name description unit quantity }
      location { id name }
      lastDetectedAtLocation { id name }
    }
    nextToken
  }
}"""

b64e = base64.b64encode(EMAIL.encode()).decode()
b64p = base64.b64encode(PASSWORD.encode()).decode()
token = requests.post(LOGIN_URL, json={"email": b64e, "password": b64p}).json()["IdToken"]

headers = {"Content-Type": "application/json", "Authorization": token}
all_items, next_token = [], None

while True:
    f = f'"{FILTER}"' if FILTER else "null"
    t = f'"{next_token}"' if next_token else "null"
    resp = requests.post(GRAPHQL_URL, json={"query": QUERY_TEMPLATE % (f, t)}, headers=headers).json()
    page = resp["data"]["inventory"]
    all_items += page["inventory"]
    next_token = page["nextToken"]
    if not next_token:
        break

def _get(d, *keys):
    for k in keys:
        d = (d or {}).get(k)
    return d

rows = []
for item in all_items:
    cp_raw = item.get("customProperties")
    try:
        cp = json.loads(cp_raw) if cp_raw and cp_raw != "{}" else {}
    except (json.JSONDecodeError, TypeError):
        cp = {}
    row = {
        "id":                 item.get("id"),
        "name":               item.get("name"),
        "description":        item.get("description"),
        "trackerSerial":      item.get("trackerSerial"),
        "state":              item.get("state"),
        "isConsumed":         item.get("isConsumed"),
        "lotNumber":          item.get("lotNumber"),
        "customerPartNumber": item.get("customerPartNumber"),
        "partNumber":         _get(item, "part", "number"),
        "partName":           _get(item, "part", "name"),
        "partUnit":           _get(item, "part", "unit"),
        "partQuantity":       _get(item, "part", "quantity"),
        "locationName":       _get(item, "location", "name"),
        "lastDetectedName":   _get(item, "lastDetectedAtLocation", "name"),
        "creationDate":       pd.to_datetime(item.get("creationDate"),    unit="ms", utc=True),
        "lastUpdatedDate":    pd.to_datetime(item.get("lastUpdatedDate"), unit="ms", utc=True),
        "expirationDate":     pd.to_datetime(item.get("expirationDate"),  unit="ms", utc=True),
        "consumedDate":       pd.to_datetime(item.get("consumedDate"),    unit="ms", utc=True),
    }
    for k, v in cp.items():
        row[f"cp_{k}"] = v
    rows.append(row)

df = pd.DataFrame(rows)
7
Select the table and load it

After the script finishes running, a window called Navigator appears. It shows a list of tables the script produced.

1. You'll see one item called df — click on it once to see a preview of the data on the right side

2. Check the checkbox next to df

3. Click the Load button in the bottom-right corner

Power BI will load all your inventory rows into a table. You'll see it appear in the Data panel on the right side of Power BI. You're done — you can now build visuals from it.

💡
To get fresh data later: click Home → Refresh in Power BI. It will re-run the script automatically and pull the latest inventory.

Item Types  — all item type definitions (part catalog)

Returns every item type with its ID, name, number, unit, quantity, and custom properties.

python 
import base64, json, requests, pandas as pd

EMAIL    = "your@email.com"
PASSWORD = "yourPassword"
FILTER   = None   # e.g. 'name == "Part A"' or None for all

LOGIN_URL   = "https://rest.api.xemelgo.com/login"
GRAPHQL_URL = "https://api.xemelgo.com/graphql"

QUERY = """
query inventoryParts($filter: String, $nextToken: String) {
  inventoryParts(input: { filter: $filter, nextToken: $nextToken }) {
    nextToken
    inventoryParts {
      id name number unit description quantity imagePath customProperties
    }
  }
}
"""

b64e = base64.b64encode(EMAIL.encode()).decode()
b64p = base64.b64encode(PASSWORD.encode()).decode()
token = requests.post(LOGIN_URL, json={"email": b64e, "password": b64p}).json()["IdToken"]

headers = {"Content-Type": "application/json", "Authorization": token}
all_items, next_token = [], None

while True:
    resp = requests.post(GRAPHQL_URL, json={
        "query": QUERY,
        "variables": {"filter": FILTER, "nextToken": next_token}
    }, headers=headers).json()
    page = resp["data"]["inventoryParts"]
    all_items += page["inventoryParts"]
    next_token = page["nextToken"]
    if not next_token:
        break

rows = []
for item in all_items:
    cp_raw = item.get("customProperties")
    try:
        cp = json.loads(cp_raw) if cp_raw and cp_raw != "{}" else {}
    except (json.JSONDecodeError, TypeError):
        cp = {}
    row = {
        "id":          item.get("id"),
        "name":        item.get("name"),
        "number":      item.get("number"),
        "unit":        item.get("unit"),
        "description": item.get("description"),
        "quantity":    item.get("quantity"),
        "imagePath":   item.get("imagePath"),
    }
    for k, v in cp.items():
        row[f"cp_{k}"] = v
    rows.append(row)

df = pd.DataFrame(rows)

idItem type identifier
name / numberDisplay name and part number
unit / quantityUnit of measure and default quantity

cp_*Expanded custom properties

Item Types at Locations  — counts and status per item type per location

Returns on-hand, missing, overdue, and due-soon counts broken down by item type and location. Great for a dashboard showing inventory health across locations.

query inventoryPartMetrics($filter: String, $nextToken: String) {
  inventoryPartMetrics(input: { filter: $filter, nextToken: $nextToken }) {
    nextToken
    inventoryParts {
      id name missingCount dueSoonCount overdueCount onHandCount totalCount
      location { id name }
      type { id name description customProperties }
    }
  }
}
"""

b64e = base64.b64encode(EMAIL.encode()).decode()
b64p = base64.b64encode(PASSWORD.encode()).decode()
token = requests.post(LOGIN_URL, json={"email": b64e, "password": b64p}).json()["IdToken"]

while True:

        "query": QUERY,

        "totalCount":      item.get("totalCount"),
        "locationId":      _get(item, "location", "id"),
        "locationName":    _get(item, "location", "name"),

Location Roles  — reference list of assignable location roles

No pagination or filter — returns the full static list in one request. Use this as a lookup table to join against the Locations query.

python 
import base64, requests, pandas as pd

EMAIL    = "your@email.com"
PASSWORD = "yourPassword"

LOGIN_URL   = "https://rest.api.xemelgo.com/login"
GRAPHQL_URL = "https://api.xemelgo.com/graphql"

QUERY = """
query locationRoles {
  locationRoles {
    roles {
      id
      name
      description
    }
  }
}
"""

b64e = base64.b64encode(EMAIL.encode()).decode()
b64p = base64.b64encode(PASSWORD.encode()).decode()
token = requests.post(LOGIN_URL, json={"email": b64e, "password": b64p}).json()["IdToken"]

Troubleshooting
!
"Python scripting is not enabled" or no Python option in Get Data

Power BI didn't find a Python installation. Go back to Step 5 and make sure the Python path is set correctly in Options → Python scripting. Close and reopen Power BI after changing it.

!
ModuleNotFoundError: No module named 'requests' (or 'pandas' or 'matplotlib')

The packages aren't installed in the Python environment Power BI is using. Run python -c "import sys; print(sys.executable)" to confirm which Python Power BI is pointing to, then run pip install requests pandas matplotlib using that same Python.

If you're using Anaconda: conda install requests pandas matplotlib in the active environment.

!
Script runs but Navigator shows no tables

Power BI only surfaces variables that are pandas.DataFrame objects. Make sure the script ends with df = pd.DataFrame(rows) and that the script didn't exit early due to an error. Check that the credentials are correct.

!
401 Unauthorized

Wrong credentials, or the token expired mid-run (tokens are valid for 480 seconds). Double-check EMAIL and PASSWORD at the top of the script, then refresh again.

!
Only getting first page of results

The API paginates via nextToken. The script loops until nextToken is null — if you truncated the script, restore the full while True loop from Step 4.
